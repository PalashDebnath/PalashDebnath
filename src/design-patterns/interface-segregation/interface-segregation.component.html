Interface Segregation Principle

The Interface Segregation Principle (ISP) states that no code should be forced to depend on methods it does not use. Interface Segregation Principle (ISP) splits interfaces that are very large into smaller and more specific ones so that clients will only have to know about the methods that are of interest to them.
