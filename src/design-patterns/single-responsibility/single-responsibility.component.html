Single Responsibility Principle

The Single Responsibility Principle (SRP) is a computer-programming principle which state that every module, class or function (method) in a computer program should have responsibility over a single part of that program's functionality, and it should encapsulate that part. On the other hand we could say a class or function (method) should have only one reason to change.
